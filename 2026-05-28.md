# 2026-05-28 工作记录

## 任务
修复 SVAC 录像回放 API (`/index/api/startPlaybackSVAC`) 无法正常播放的问题。

## 问题现象
调用 `startPlaybackSVAC` API 回放 `.rtp` 录像文件到目标地址 `192.168.9.255:6000` 时，接收端无法收到 UDP 数据包，无画面。

## 根因分析 & 修复

### 修复1：UDP 广播 SO_BROADCAST 未设置
- **文件**: `src/Player/PlaybackSVAC.cpp`
- **问题**: 目的地址 `192.168.9.255` 是广播地址，UDP socket 默认不允许发广播包，`sendto()` 返回 `EACCES`
- **修复**: 在 `onConnect()` 中 UDP 模式下添加 `SockUtil::setBroadcast(_socket_rtp->rawFD())`

### 修复2：RTP 时间戳回绕检测 unsigned 溢出
- **文件**: `src/Player/PlaybackSVAC.cpp` — `indexRtpFile()`
- **问题**: 原始代码 `stamp < prev_stamp - 0x80000000u` 中，当 `prev_stamp < 0x80000000` 时，无符号减法溢出成超大值，导致每个包都误判为回绕，`wrap_offset` 被累加数百次，`stamp_ms` 计算结果达到 5 年级别
- **修复**: 改为标准 RTP 回绕检测：`prev_stamp > stamp && (prev_stamp - stamp) > 0x80000000u`

### 修复3：SO_BROADCAST 调用时机错误
- **文件**: `src/Player/PlaybackSVAC.cpp` — `startUdpActive()`
- **问题**: `setBroadcast` 在 `onConnect()` 中调用，但 `bindPeerAddr`（底层 connect()）在此之前执行。Windows 要求 `SO_BROADCAST` 必须在 `connect()` 之前设置，否则连接到广播地址会失败
- **修复**: 在 `startUdpActive()` 中，`bindPeerAddr` 之前添加 `SockUtil::setBroadcast()`

### 修复4：异步回调中 weak_ptr 导致对象提前析构
- **文件**: `src/Player/PlaybackSVAC.cpp` — `startUdpActive()`
- **问题**: `startUdpActive()` 通过 `WorkThreadPool` 异步执行 DNS + socket 初始化。若异步 lambda 捕获 `weak_ptr`，API 处理函数返回后 local `shared_ptr` 销毁，对象先析构（触发 `~PlaybackSVAC()` → `stop()`），异步回调再执行时 `weak_ptr.lock()` 返回 null，HTTP 响应永远不会发送
- **修复**: 将异步 lambda 中的 `weak_ptr` 全部改为 `shared_ptr`（`auto self = shared_from_this()`），确保对象在异步完成前存活

### 修复5：BufferRaw::create() 未设置 _size 导致 BufferOffset 断言失败
- **文件**: `src/Player/PlaybackSVAC.cpp` — `feedNextRtp()`
- **问题**: `BufferRaw::create(size)` 只调用 `setCapacity(size)` 分配内存，不设置 `_size`，导致 `buffer->size()` 返回 0。`BufferRtp`（即 `BufferOffset<Buffer::Ptr>`）内部执行 `assert(offset + size <= max_size)` → `assert(4 + 0 <= 0)` → 断言失败抛异常
- **修复**: 改用 `BufferLikeString` + `std::string` 初始化方式，底层 `std::string` 的 `size()` 天然等于分配的容量

### 附加修改
- `onFlushRtpList()` 中 `send()` 调用添加 try-catch + 调试日志，捕获异常时打印具体错误信息
- `onFlushRtpList()` 中发送前添加 `send packet size=X offset=4` 日志便于排查
